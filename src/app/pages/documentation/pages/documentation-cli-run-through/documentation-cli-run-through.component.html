Once you've installed our CLI, you're set to go ! We use the classical following pattern for our commands : For instance : Note options can generally be used as double dash + full name or simple dash with alias At any moment, you can get help about spm or its command by typing spm, spm <model> or spm <model> <action> with no argument, --help or -h option.
The CLI offers the following models at the moment:
model description
user used to register, login, display current user information (excepted password) and logout
module used to interact with spm core features
version used to display current spm CLI version
The following models are currently being implemented:
command description
project used to interact with projects, large ensemble of codes used to deliver a front-end application, bringing powerful enterprise-focused features
styleguide used to share and enable a common graphical chart and best practices
The next sections will focus around spm modules, with the following commands:
command description
admin to modify permissions
clone to download source files used to make a publication
create to tag a folder as a spm module and generate spm files
detail to display spm-related information
edit to modify spm-related information
generate to create a new instance with customized parameters
install to use a module inside your code
list to list spm modules used in your code
publish to send a module to spm registry
search to display spm modules matching specific criteria
unpublish to remove a module from spm registry
version to update your module's version
All previous module commands are described in next sections.
