Once you've installed our CLI, you're set to go ! We use the classical following pattern for our commands : For instance : Note options can generally be used as double dash + full name or simple dash with alias At any moment, you can get help about spm or its command by typing either spm with no argument or spm with a command and the --help or -h option.
The CLI offers the following commands at the moment :
command description
user used to register, login or display current user information (excepted password)
init creates a package-spm.json tracking file to optimize one project's portability, versionning and unlock powerful spm features
install downloads a graphical package of your choice in your project
generate customizes a package to create your own style
use imports for you a style in your project
ls displays all dependencies of your project
version to update your project's spm version before publication
publish makes your project available for download to a community of your choice
owner to grant contribution rights on a package you're contributing to
All commands above (except the user and the owner one) are described in next sections.
