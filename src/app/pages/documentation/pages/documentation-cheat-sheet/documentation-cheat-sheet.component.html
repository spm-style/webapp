User authentification

register, login and logout

$ spm user

Opens a prompter to register or login with spm

$ spm user logout

disconnects you from spm CLI

$ spm user info

displays current information on the logged user

Launch a project

initializes core information on your project to enable powerful spm features

$ spm init

Opens a prompter where you can fill your project's basic information

Install wonderful packages

Once you've found a packge you like, add it to your project

$ spm i

Alias for spm install

$ spm install

to install all spm dependencies saved in your project

$ spm install package1 package2

to install the latest versions of package1 and package2

$ spm install packageA@2.0.8

to install the version 2.0.8 of packageA

$ spm install packageA --save

to install packageA and save it as a dependency of your project

Customize your style

You can modify specific parameters of package you've installed

$ spm g

Alias for spm generate

$ spm generate packageA customName

opens a prompter which will help you customize packageA's properties and generates a customized instance called customName

$ spm generate packageA customName --rename

to select all classes' names when you generate a custom instance

$ spm generate packageA customName --use

to generate a custom instance and use it in your main stylesheet

$ spm generate packageA customName --save

to generate a custom instance and save it as a dependency of your project

Use your style in your project's stylesheets

To use a style, your stylesheet must import it. Don't waste time figuring out the relative path.

$ spm u

Alias for spm use

$ spm use customName

imports the style from the instance customName

$ spm use customName --save

import an instance's style in your project and save it as one of your project's dependencies

Inspect your project

Some projects can import many graphical component : display their tree list.

$ spm ls

displays the tree list of all dependencies used and their saved instances

Updates your project version

To track your project, maintain it and manage teams, versionnng is powerful. This increments your version.

$ spm v

Alias for spm version

$ spm version minor

a version is a 3 digit object : MAJOR.MINOR.PATCH. Use one of those (case insensitive) to udpate your project's version

Publish your project

This adds your project to our registry, allow you to share it and access is from anywhere.

$ spm p

Alias for spm publish

$ spm publish

to publish your project

$ spm publish myPackage@2.0.0

to publish your project as the version 2.0.0

Manage project's contributors

to add, remove contributors from project you're already contributing to, and display the list of contributors

$ spm owner add user packageName

to add a user (email or login) as a package's contributor

$ spm owner rm user packageName

to remove a user (email or login) from the contributors of a package

$ spm owner ls packageName

To display logins and emails of contributors to a package
