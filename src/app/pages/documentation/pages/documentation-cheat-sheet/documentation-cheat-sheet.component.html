User authentification

register, login and logout

$ spm user register

Opens a prompter to register with spm

$ spm user login

connects you to spm CLI

$ spm user detail

displays current information on the logged user

$ spm user logout

disconnects your user from spm

Start a module

initializes core information on your module to enable powerful spm features

$ spm module create

Opens a prompter where you can fill your module's basic information

Install wonderful modules

Once you've found a module you like, add it to your code

$ spm m i

Alias for spm module install

$ spm module install

to install all spm dependencies saved in your code (refer to module-spm.json)

$ spm module install module1 module2

to install the latest versions of module1 and module2

$ spm module install moduleA@2.0.8

to install moduleA's version 2.0.8

$ spm module install moduleA --save

to install moduleA and save it as a dependency of your code

Customize your style

You can modify specific parameters of modules you've installed

$ spm m g

Alias for spm module generate

$ spm module generate customName --module-name moduleA

opens a prompter which will help you customize moduleA's properties and generates a customized instance called customName

$ spm module generate customName --module-name moduleA --classes-rename

to select all classes' names when you generate a custom instance

$ spm module generate customName --module-name moduleA --assign myInstance

to generate a custom instance and store the javascript instance object in a myInstance variable

Inspect your project

Some codes can import many graphical component : display their tree list.

$ spm module list

displays the tree list of all dependencies used and their saved instances

Updates your module version

To track your module, maintain it and manage teams, versionnng is powerful. This increments your version.

$ spm m v

Alias for spm module version

$ spm module version minor

a version is a 3 digit object : MAJOR.MINOR.PATCH. Use one of those (case insensitive) to udpate your module's version

Publish your module

This adds your module to our registry, allow you to share it and access is from anywhere.

$ spm m p

Alias for spm module publish

$ spm module publish

to publish your module

$ spm publish myModule@2.0.0

to publish your module as the version 2.0.0

Manage module's contributors

to add, remove contributors from a module you're already contributing to, and display the list of contributors

$ spm module admin --owner-add axolotl

to add a user whose email or login is axolotl as a module's contributor

$ spm module admin --owner-remove axolotl

to remove a user (email or login) from the contributors of a module

$ spm module admin --owner-list

To display logins and emails of contributors to a module
