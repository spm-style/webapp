Before publishing a module, you must have a spm profile. Once your code is ready, you can publish your module using the spm publish or short spm p CLI command. style version description author jsStandard files license repository readme keywords mainClass category responsiveness If you have no html index file or if the file is empty, spm will browse the folders recursively to analyze where the main class is used and will suggest reference DOMs based on what we found. We check your publication's style does not affect any external element. For this reason, all your style selectors should be driven by your main class and/or your project's class No file out of your current folder will be sent to our registry, therefore we check you don't import anything from local sources out of your module folder. Instance variables always start with $_. Any variable starting with this prefix will be considered as instance variable. Make sure your javascript instances variables are always declared and instantiated at once. Make sure your stylsheet instances variables are declared and instantiated in the variables-spm.scss file. Your module's script will be processed into a javascript class. Any variable starting with $$_ will be considered as an export variable, and will be a property of any instance of the class. Keep in mind that this code will be executed in strict mode. Wether the publication suceeds or not, a report will be displayed in your terminal. To ensure each module's unicity and the compatibility between all modules, spm renames all publications based on this model : *author*_*name*. The classes and variables will be renamed the same way. anyone if it is a public publication specific collaborators you've chosen if it is a private publication Since spm processes your code to make it accessible to our great community, we keep the source files you sent us. Use CLI command spm module clone <moduleName> to download them back.
