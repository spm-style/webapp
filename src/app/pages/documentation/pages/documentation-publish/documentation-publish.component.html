Publish your creation

Before publishing a package, you must have a spm profile. Once your code is ready, you can publish your package using the spm publish or short spm p CLI command. style type version description author entry scripts license repository readme keywords main category classes responsiveness Moreover, you need to have a ref-dom.html template file located at the root of your publication if you don't use one of spm templates. We check your publication's style does not affect any external element. For this reason, all your style selectors should be driven by your main class and/or your project's class Wether the publication suceeds or not, a report will be displayed in your terminal. To ensure each package's unicity and the compatibility between all packages, spm renames all publications based on this model : *author*_*name*. The classes and variables will be renamed the same way. anyone if it is a public publication specific collaborators you've chosen if it is a private publication
