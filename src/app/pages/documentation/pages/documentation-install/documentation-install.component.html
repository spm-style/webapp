Install fancy packages

Let's say you want to use the graphical element you've seen in package `apollo_onOff`. Open your terminal and navigate to your project, then use the spm install or short spm i CLI command. At the end of the installation, the CLI displays the detail of what was installed, and WARNING / ERROR messages : The following package has been installed: By default, spm installer optimizes the quantity of information installed in your computer. If the requested version of your package already exists in your project or in you registry, no additional code will be written and a symbolic link will be created. You can specify multiple arguments to install multiple packages. The following package has been installed: If no argument is specified, spm will install all project's dependencies. To install all dependencies listed in a project's `package-spm.json`, just use `spm i` command without any argument.

local installation

You can precise that the package has to be installed locally with option --local or -l.

global installation

You can do the same forcing the package to be installed in your own global registry, using option --registry or -r. Note that `sudo` command may be required.

target a specific version

If you want, you can precise the version of the package you want to install using `@` marker: If no version is precised, the Long Term Support (LTS) will be automatically selected.

save package as a dependency

If you want to save this package in your project so you can easily share its content, you can use option --save or -s. It will add the version of this package in `package-spm.json` file's dependencies key :

reinstall a package

Since spm doesn't download files already existing in your registry or project, you can force the re-installation by using option --force or -f.
