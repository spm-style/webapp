Let's say you want to use the graphical element you've seen in module `apollo_onOff`. Open your terminal and navigate to your project, then use the spm module install or short spm m i CLI command. At the end of the installation, the CLI displays the detail of what was installed, and WARNING / ERROR messages : The following module has been installed: By default, spm installer optimizes the quantity of information installed in your computer. If the requested version of your module already exists in your project or in you registry, no additional code will be written and a symbolic link will be created. any installed module will generate instances files if they aren't already in your code. Instances code will be added in both the stylesheet and the script. spm classes always start with an upper case letter, as per convention. Any javascript exported value can be accessed as one key of the assigned variable object. For example, if the Apollo-onOff class exports the moon variable, it is stored in apollo_onOff.moon in the code above. The instances are imported in the instance stylesheet as well, using scss mixins. If your style is css, don't be scared, everything is implemented in the spm_instances/.spm_instances.css file. using both spm_instances files makes it really easy to customize and test different implementations of modules. Give it a try ! You can specify multiple arguments to install multiple modules. The following modules have been installed: If no argument is specified, spm will install all project's dependencies. To install all dependencies listed in a project's `module-spm.json`, just use `spm m i` command without any argument.

local installation

You can precise that the module has to be installed locally with option --local or -l.

global installation

You can do the same forcing the module to be installed in your own global registry, using option --registry or -r. Note that `sudo` command may be required.

target a specific version

If you want, you can precise the version of the module you want to install using `@` marker: If no version is precised, the latest version will be automatically selected.

save module as a dependency

If you want to save this module in your project so you can easily share its content, you can use option --save or -s. It will add the version of this module in `module-spm.json` file's dependencies key :

reinstall a module

Since spm doesn't download files already existing in your registry or project, you can force the re-installation by using option --force or -f.
