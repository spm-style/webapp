Initializing your module adds a module-spm.json file at the level where you enter the command spm module create CLI command. It opens a prompter asking for the module's main parameters : folder myModule file myModule/variables-spm.scssfor the scss instance variables, prefixed by $_ file myModule/myModule.js for the module script, embedding js instance variables prefixed by $_ and implementing export variables, prefixed by $$_ file myModule/myModule.html for the reference DOM file myModule/module-spm.json for the module json configuration file file myModule/myModule.scss for the main stylesheet, which can import other stylesheets The following options are available for module creation:
option description
--version <version> the module's version
--style <style> chose between scss and css (default) stylesheet language
--main-class <mainClass> the main class, wrapping all the module behaviour in reference DOM and stylesheet
--description <description> your module's description
--js-standard <standard> legacy (ES5) or modular(ES6): legacy is defined by default since modular is not supported by all browsers yet
--category <category> your module's category (e.g. carousel, input, ...)
--responsive <devices> the devices your module is compatible with
--keywords <keywords> a couple of words used for referencing purposes
--readme <readmeFile> the readme filename
--repository <repository> the git repository where the module is hosted
--license <license> intellectual property
--classes <classes> all the DOM classes your project implements
--html-name <htmlName> the html entry file's name
--ss-name <ssName> the (s)css entry file's name
--js-name <jsName> the js entry file's name
--no-html-file to prevent the html file creation
--no-ss-file to prevent the (s)css file creation
--no-js-file to prevent the js file creation
--no-file no file is created
--flat the module is created in the current folder, and no folder will be created
--default no prompter opened, default information used
--force forces the creation of files or folders if they already exist
--debug displays the debug logs
-h, --help information about spm module creation feature in the CLI
Most of the features described above require an entry file to run more smoothly. You cannot publish a module which has not been initialized. Which means you cannot use spm to share, demonstrate or version your code.
