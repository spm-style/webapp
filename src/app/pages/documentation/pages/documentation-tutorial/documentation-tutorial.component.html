If you're looking for a step-by-step guide, you're at the right place. Once you have installed our CLI, you will want to be able to install modules as well as creating and publishing your own one. This section covers how to do that.

Browse modules

In this tutorial, we are looking for a design for the main title of our current or future web project. We go on spm-style.com page and browse to the modules section. We can use the search bar with the “title” word to target specific modules used for titles for instance. We can click on the module image to have more information about it. This detailed page is called sandbox : it is very useful to see how we want to customize and use the module. We can simulate the device on which the component is used (1), change the zoom value (2), download source files directly (3), chose the active classes (4), modify variables (5) or check for the html code we can use to add it to our code (6). Don't hesitate to browse this section to find a module fitting your needs.

Create a spm project or configure an existing project as a spm project

First, we create or use an existing folder and navigate inside it. You can name it the way you want, we will call it myProject Then we use the spm command to create a project We will use the following values:project name set as myProjectdefault style set as cssproject's main html file set as index.htmlproject's main script set as script.jsproject's main stylesheet set as style.cssproject's styleguide set as styleguide.cssdescription set as this is my first project with spmjs standard set as legacy spm makes an overview of the chosen parameters. Type y to accept. The following message should be displayed in your console: file /Users/username/Desktop/myProject/index.html successfully createdfile /Users/username/Desktop/myProject/script.js successfully createdfile /Users/username/Desktop/myProject/style.css successfully createdfile /Users/username/Desktop/myProject/project-spm.json successfully createdfile /Users/username/Desktop/myProject/styleguide.css successfully createdfile /Users/username/Desktop/myProject/environment.js successfully createdfile /Users/username/Desktop/myProject/variables-spm.css successfully created If we open our main html file in our browser, we'll see something like this :

Modify directly the instances Files (advanced usage)

As we’ve seen in the previous section, the result of the command was the update of the two instances files, using the same constructor with different parameters. We can do the same manually. Note that we will need to compile the spm_instances.scss file into .spm_instances.css (notice the dot at the beginning of the css file) at the end of the manual modification.
